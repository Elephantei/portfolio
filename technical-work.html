Scaling High-Throughput Workloads on EKS

Problem

A high-frequency trading platform was experiencing unpredictable traffic spikes during market volatility, leading to:

  • Service degradation during peak trading hours
  • Inconsistent response times affecting trade execution
  • Manual scaling interventions causing operational overhead
  • Resource contention between different workload types

Approach

  • EKS Cluster Management: Implemented multi-AZ cluster architecture with node groups optimized for different workload types
  • Autoscaling Policies: Configured HPA and VPA with custom metrics, fine-tuned scaling policies for rapid response to traffic spikes
  • Namespace Isolation: Split workloads across dedicated namespaces to prevent resource contention and enable independent scaling
  • Istio Service Mesh: Implemented traffic management, security policies, and observability across microservices
  • GitLab CI/CD: Built fully automated pipelines with integrated testing, security scanning, and deployment automation
  • Pod Disruption Budgets: Enhanced PDBs to ensure zero-downtime deployments during rolling updates

Tools Used

Kubernetes EKS Helm Istio GitLab CI/CD Prometheus Grafana Terraform

Outcome

  • 99.9% uptime during high-traffic periods (previously 95.2%)
  • 30% improvement in p95 latency during peak loads
  • Zero manual interventions required for scaling events
  • 40% reduction in infrastructure costs through optimized resource allocation
  • Successfully handled 10x traffic spikes without service degradation

Terraform Modules for Multi-Squad Adoption

Problem

Four development teams were creating inconsistent Azure infrastructure across environments, leading to:

  • Duplicated Terraform code and configuration drift between teams
  • Security vulnerabilities from inconsistent NSG configurations and Docker CVEs
  • Deployment failures due to environment-specific hardcoded values
  • Increased maintenance overhead and knowledge silos

Approach

  • Module Architecture: Designed reusable Terraform modules with clear interfaces and versioning strategy
  • Standardisation: Created modules for common Azure patterns (VNet, AKS, SQL Database, Application Gateway) with built-in security best practices
  • Documentation: Built comprehensive module documentation with examples and usage patterns
  • CI/CD Integration: Implemented automated testing and validation pipelines for module changes
  • Governance: Established module registry and approval process for production deployments
  • Embedded Onboarding: Created self-service onboarding process where users add themselves to Azure groups directly from the repository

Tools Used

Terraform Azure Azure DevOps Checkov Terraform Cloud Git

Outcome

  • 80% reduction in infrastructure provisioning time across teams
  • Significant reduction in security vulnerabilities through standardized NSG configurations
  • 95% code reuse across different environments and teams
  • Self-service onboarding reduced manual setup time from days to minutes
  • Successfully migrated 4 development teams to standardized module approach
